## 一、系统总体架构与设计愿景

本项目致力于从零手写并打造一款遵循 **LoongArch32 精简版指令集（LA32R / 89 条基准指令）** 的工业级高频可综合 SoC 处理器系统。

### 核心架构要点：
* **指令集架构 (ISA)**：LoongArch32（完整支持初赛规定的 89 条整型、移位、逻辑、乘除法及分支跳转指令）。
* **微架构演进 (Microarchitecture)**：
  * 第一阶段：最小完备**单周期 CPU 架构**（建立核心数据通路与基本控制器，跑通算法程序）；
  * 第二阶段：经典**五级流水线架构**（IF, ID, EX, MEM, WB），包含完整的前递单元（Forwarding Unit）与暂停冲刷逻辑（Hazard/Stall Unit），解决 `Load-Use` 与分支控制冲突；
  * 第三阶段：高性能**存储与总线架构**，挂载 8KB Direct-Mapped L1 I-Cache 与 8KB D-Cache，接入标准 AMBA **AXI4 / AXI4-Lite** 总线互联架构。
* **目标上板平台**：Xilinx Artix-7 FPGA 开发板（使用 Vivado 2023.2 及以上版本综合实现，目标主频 80MHz+）。

## 三、工业级硬件代码编码铁律

为了让我们的 SoC 能够在 FPGA 上顺利综合、不产生致命的锁存器或死锁，全体成员必须遵守以下规范：

### 3.1 命名规范 (Naming Conventions)
1. **模块与文件名一致**：模块名全小写、下划线分隔，且必须与文件名完全一致（如 `module regfile` 对应文件 `regfile.v`）。
2. **端口时钟与复位**：系统时钟统一命名为 `clk`；低电平异步复位信号统一命名为 `rst_n`（0 为复位有效，1 为正常工作）。
3. **接口信号前缀/后缀**：
   * 输入端口以 `_i` 结尾，输出端口以 `_o` 结尾（或严格用 `cpu_req`, `bus_rdata` 等具备语义的方向名）；
   * 寄存器触发器变量命名明确其时序属性；
   * 严禁使用毫无意义的 `a, b, c, temp, test` 等作为模块跨层传输接口！

### 3.2 逻辑设计排雷“五戒” (The 5 Commandments)
1. **严禁多驱动赋值 (Multi-Driver Assignment)**：在一个 `.v` 文件中，**绝对不允许**两个不同的 `always` 块或者 `assign` 语句同时驱动同一个信号！谁驱动谁维护，责任明确。
2. **严防隐形锁存器 (Latch Preventing)**：
   * 在使用 `always @(*)` 编写组合逻辑的 `if-else` 或 `case` 语句时，**必须补齐所有条件分支（写完 `else` 和 `default`）**！或者在 `always` 块第一行赋初值！
   * 时序逻辑使用非阻塞赋值 `<=`；组合逻辑严格使用阻塞赋值 `=`！**决不能在同一个 `always` 块中混用 `=` 与 `<=`！**
3. **AXI 总线握手铁律 (No Handshake Deadlock)**：
   * 作为主设备发出 `VALID` 信号时，**绝不能依赖于检测到对方 `READY == 1` 才拉高**！一旦拉高 `VALID`，必须保持不变直到 `READY == 1` 同步握手成功！
4. **杜绝组合逻辑环 (No Combinational Loops)**：信号不能不用时序寄存器打拍就通过逻辑门直接自我反馈（比如 `assign a = a & b;`），这会导致芯片物理上发生振荡自毁。
5. **异步复位，同步释放 (Asynchronous Reset, Synchronous Release)**：系统顶层必须对外部输入的复位信号做两级触发器同步打拍处理后，再分发给各个核心子模块，防止亚稳态！

## 五、总体阶段安排与任务拆解

本节为全体团队成员的**核心作战地图**。研发工程采用分阶段里程碑形式实施，任何模块写完必须通过验证组验收才能进入下一阶段。

---

### 🚩 Phase 0：工程启动、自动化环境与基础架构落地
**【阶段目标】**：统一全员思想，打通从 RTL 编译、仿真到波形查看的自动化研发流水线。
* **主要任务**：
  1. **目录建设与规范配置**：全体队员拉取最新工程树，熟悉 `rtl/`, `tb/`, `sim/`, `fpga/`, `doc/` 各目录职责；
  2. **验证环境打通**：在各个开发机（Linux / Windows）上调通 `Verilator + C++ / Cocotb` 或 `Vivado Sim` 命令行平台，确保能通过 `Makefile` 或脚本运行一键回归；
  3. **硬件思维内训**：研读 `doc/` 下的架构与排雷手册，彻底摒弃 C 语言时间串行思维，掌握时钟触发器与组合逻辑的物理意义。
* **验收指标**：团队每位成员能够独立运行一键编译仿真命令，并在波形查看器（GTKWave / VSCode / Vivado）中成功抓取测试时钟与复位信号。

---

### 🚩 Phase 1：单周期完备核心与 15 条基础指令闭环
**【阶段目标】**：从零构建可综合的通用数据通路（Data Path）与主控制单元（Controller），跑通基础数学运算汇编小程序。
* **主要任务**：
  1. **通用寄存器堆 `regfile.v`**：实现 32x32 双读单写端口，严格保证 `$r0` 恒定为零，异步复位有效；
  2. **核心算术逻辑单元 `alu.v`**：实现基础加减法、逻辑运算（AND/OR/XOR/NOR）与大小比较运算；
  3. **程序计数器 `pc.v` 与主控制器 `controller.v`**：结合操作码（Opcode）译码真值表，建立单周期取指、译码、执行与写回的数据流通路；
  4. **15 条首批基准指令闭环**：打通 `add.w`, `sub.w`, `addi.w`, `lu12i.w`, `slt`, `sltu`, `and`, `or`, `xor`, `ld.w`, `st.w`, `beq`, `bne`, `b`, `jirl`。
* **验收指标**：编写计算 $1+2+3+...+100$ 及斐波那契数列的汇编小程序，通过 `hex` 文件载入自测内存，计算结果准确无误写入寄存器，全历程仿真波形无未知态 `X` 和高阻态 `Z`！

---

### 🚩 Phase 2：五级流水线架构重构与 89 条基准指令全覆盖
**【阶段目标】**：将单周期重构为工业级五级流水线，攻克冒险冲突处理，完成初赛全部基本指令要求。
* **主要任务**：
  1. **流水线寄存器切分**：构建 IF, ID, EX, MEM, WB 五个物理流水段，增加级联寄存器（`if_id.v`, `id_ex.v`, `ex_mem.v`, `mem_wb.v`）；
  2. **硬件冒险处理单元 `hazard.v`**：
     * **数据冒险前递 (Forwarding Unit)**：将 EX 段、MEM 段及 WB 段的最新结果直接前递回 ID/EX 运算端；
     * **流水线暂停 (Stall/Freeze)**：检测 `Load-Use` 冲突及跳转指令控制冲突，精确插入气泡并保持 PC/IF/ID 寄存器不动；
  3. **89 条基准指令全覆盖**：扩充译码器，集成逻辑移位及原生乘除法器 IP（支持同步运算），覆盖初赛考核指南中全部 89 条指令；
  4. **自动化回归测试**：导入组委会第一阶段初赛基准回归测试集。
* **验收指标**：在测试平台上连续运行 89 项基础指令功能测试小程序，达到 **89 项测试安全通过（100 分满分）**！无任何死锁与冲突误冲刷。

---

### 🚩 Phase 3：存储子系统接入与 AXI4 总线桥接挂载
**【阶段目标】**：摆脱简易内存模型，实现真正可综合的 L1 Cache 与标准 AMBA AXI 总线互联。
* **主要任务**：
  1. **L1 指令缓存 `icache.v`**：设计 8KB 直接映射（Direct-Mapped） Cache 状态机（IDLE -> LOOKUP -> REFILL 总线补行）；
  2. **L1 数据缓存 `dcache.v`**：设计 8KB D-Cache 状态机，支持 Write-Through + Write Buffer（或 Write-Back）策略；
  3. **AXI4 总线桥接单元 `axi_master_if.v`**：将 Cache 后端访存转化为标准 AXI4 AR/R/AW/W/B 通道时序，严格遵守 **`VALID` 不依赖 `READY`** 的工业界握手铁律；
  4. **SoC 官方外设对接**：通过多主多从交叉开关（Crossbar），对接组委会提供的 DDR3 内存控制器、SPI Flash 存储与 RS232 串口等外设 IP。
* **验收指标**：CPU 核心依靠 AXI 总线和 Cache 成功自举，顺利从总线外存载入复杂程序运行，并通过 UART 串口正常向终端打印中英文字符日志！

---

### 🚩 Phase 4：Vivado FPGA 物理上板、时序优化与跑分冲刺
**【阶段目标】**：实现物理芯片落地，消除时序违例，在高频运行下冲刺初赛/决赛最高评测分数。
* **主要任务**：
  1. **Vivado 工程上板综合**：导入板载 XDC 物理引脚约束与时钟约束文件，开启 Synthesis 与 Implementation 流程；
  2. **时序收敛与拆解优化 (Timing Closure)**：分析综合报告中的最差负时钟裕量（WNS < 0），通过拆解长组合逻辑链、优化多路选择器、拆解高扇出信号等手段，保证所有时序裕量为正（WNS >= 0）；
  3. **Artix-7 FPGA 物理烧录**：将生成的 `.bit` 比特流文件烧入真实的 Xilinx Artix-7 开发板中，观察板载状态数码管与调试 LED；
  4. **性能评估与主频冲高**：运行系统计数器及大赛官方性能评测跑分套件（如 Dhrystone、CoreMark），定位 IPC 瓶颈并尝试将工作主频从 **60MHz 向 80MHz、100MHz+** 发起极速冲刺！
* **验收指标**：实物开发板上电运行稳定，长时间跑分测试不异常死机、不掉线，串口持续高速打印优秀的评测报告跑分结果，锁定全国大赛决赛首批晋级门票！🏆
